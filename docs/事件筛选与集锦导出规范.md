# 事件筛选与排序规则 v1 + 集锦导出规范 v1

## 文档信息
- 版本：v1
- 依据：`Team Interface Contract v1.2 Frozen` §3.4（D 模块责任）、§8.6（E→D 集锦任务）、§8.7（record_index.json 结构）、§11 联调顺序第五步
- 适用模块：highlight-generation-module（事件与集锦生成模块，模块 D）
- 输入模块：vision-event-module（模块 C，候选事件）、record-program-module（模块 B，录像文件）
- 事件类型：仅 4 类（合同 §8.5 冻结）：`goal_candidate`、`shot_candidate`、`danger_attack_candidate`、`celebration_candidate`
- 集锦模式：MVP 第一版仅做全场精彩集锦（合同 §3.4 强制要求 #1）
- 冻结要求：事件类型、优先级、裁剪缓冲、输出格式一经冻结不得大改

---

## 1. 概述

事件与集锦生成模块负责将视觉模块输出的候选事件与录像文件拼接为全场精彩集锦。MVP 第一版仅支持"全场精彩集锦"模式，不生成个人集锦。

---

## 2. 事件类型与优先级

### 2.1 事件类型枚举（冻结）

| 事件类型 | 含义 | 优先级 | 说明 |
|------|------|------|------|
| `goal_candidate` | 进球候选 | 最高 | 球接近球门且画面节奏剧变 |
| `shot_candidate` | 射门候选 | 高 | 明显射门动作 |
| `danger_attack_candidate` | 危险进攻候选 | 中 | 门前/禁区附近明显攻势 |
| `celebration_candidate` | 庆祝候选 | 低 | 进球后庆祝动作群聚 |

### 2.2 优先级排序

排序权重公式（score 越高越排前）：
```
score = event_priority_weight × confidence
```

优先级权重表：

| 事件类型 | 权重 |
|------|------|
| `goal_candidate` | 100 |
| `shot_candidate` | 60 |
| `danger_attack_candidate` | 40 |
| `celebration_candidate` | 20 |

排序规则：
1. 按 score 从高到低排列
2. score 相同时按时间先后排列
3. 若总集锦时长超限，从 score 最低的事件开始裁减

---

## 3. 事件去重规则

### 3.1 时间轴去重

两个事件时间窗重叠超过 50% 时，视为重复：

| 场景 | 处理 |
|------|------|
| goal 与 shot 重叠 | 保留 goal，删除 shot |
| shot 与 danger_attack 重叠 | 保留 shot，删除 danger_attack |
| 同类型事件重叠 > 50% | 保留置信度更高的 |

### 3.2 celebration 特殊处理

- celebration 之后 20 秒内判断是否有前序 goal/shot，若有则附加到前序事件后
- 独立 celebration（无关联 goal）仅在集锦时长充足时保留

---

## 4. 视频裁切缓冲参数

### 4.1 片段缓冲时间（合同 §8.6 冻结值）

| 事件类型 | 提前 (pre_sec) | 延后 (post_sec) |
|------|------|------|
| `goal_candidate` | 8 秒 | 10 秒 |
| `shot_candidate` | 6 秒 | 6 秒 |
| `danger_attack_candidate` | 5 秒 | 5 秒 |
| `celebration_candidate` | 3 秒 | 8 秒 |

### 4.2 裁切约束

- 片段起始不得早于录像起始时间，不得晚于录像结束时间
- 片段最小长度：3 秒（合并所有相邻事件缓冲区后的片段）
- 相邻片段间隔 < 3 秒时，自动合并为一段

---

## 5. 集锦总时长控制

### 5.1 策略

| 参数 | 默认值 | 说明 |
|------|------|------|
| max_total_duration_sec | 600 | 集锦最长 10 分钟 |
| max_event_count | 15 | 最多选取 15 个事件片段 |

### 5.2 时长预算分配

1. 先行填充所有 goal_candidate（最高优先级）
2. 按 score 顺序填充 shot_candidate
3. 按 score 顺序填充 danger_attack_candidate
4. 剩余时长填充 celebration_candidate
5. 每步填充后检查总时长是否超限

### 5.3 裁减策略

若总时长超限：
- 从优先级最低的 celebration 开始删减
- 同优先级内从 score 最低的删减
- 至少保留 1 个 goal_candidate（如果存在）

---

## 6. 集锦导出规范

### 6.1 输出文件

| 参数 | 冻结值 |
|------|------|
| 文件路径 | `{data_root}\output\{match_id}\full_highlight.mp4` |
| 容器 | `.mp4` |
| 视频编码 | H.264 |
| 分辨率 | 1920×1080 |
| 帧率 | 25 fps |

### 6.2 片段拼接

- 使用无损拼接（copy 码流），避免二次编码降低画质
- 若源视频分辨率不一致（不应出现，但防御），统一缩放到 1920×1080
- 片段之间 0 秒黑场过渡（硬切，不做特效）

### 6.3 音轨处理

MVP 阶段：
- 比赛录像通常无声（工业相机无音频采集）
- 若存在音轨，静音处理，不保留

### 6.4 片头片尾

MVP 阶段不做片头片尾，直接进入赛事内容。

### 7.1 record_index.json

由录像模块输出，集锦模块读取：

```json
{
  "match_id": "match_20260405_001",
  "record_start_timestamp_ms": 1712323200000,
  "record_end_timestamp_ms": 1712328600000,
  "duration_sec": 5400,
  "video_codec": "h264",
  "container": "mp4",
  "fps": 25,
  "source_resolution": "5mp_native",
  "program_resolution": "1920x1080",
  "cam_01_raw_path": "{data_root}\\raw\\match_20260405_001\\cam_01.mp4",
  "cam_02_raw_path": "{data_root}\\raw\\match_20260405_001\\cam_02.mp4",
  "cam_01_cut_path": "{data_root}\\program\\match_20260405_001\\cam_01_cut.mp4",
  "cam_02_cut_path": "{data_root}\\program\\match_20260405_001\\cam_02_cut.mp4",
  "program_path": "{data_root}\\program\\match_20260405_001\\program.mp4",
  "status": "success"
}
```

    {
      "event_id": "evt_003",

### 7.3 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `event_id` | string | 事件唯一 ID |
| `event_type` | string | 事件类型枚举值 |
| `confidence` | float | 置信度 [0,1] |
| `start_sec` | float | 事件起始时间（秒，相对于录像起始） |
| `end_sec` | float | 事件结束时间（秒，相对于录像起始） |
| `camera_id` | string | 事件发生的机位 |

## 8. 集锦生成流程

```
1. 读取 record_index.json → 确认录像文件路径
2. 读取 event_candidates.json → 解析事件列表
3. 去重：时间重叠事件合并或删除
4. 排序：按 score 排列
5. 添加缓冲：每段 pre_sec + post_sec
6. 合并相邻片段（间隔 < 3s 的）
7. 时长裁剪：超限时段落从低优先级开始删减
8. 从录像文件中裁切视频片段
9. 拼接片段 → full_highlight.mp4
10. 上报任务状态给平台模块
```

## 9. 接口

### 9.1 平台模块 → 集锦模块：生成集锦

**方法**：`POST`
**路径**：`/api/v1/highlight/matches/{match_id}/generate`

请求体：
```json
{
  "match_id": "match_20260405_001",
  "mode": "full_highlight",
  "record_index_path": "{data_root}\\metadata\\{match_id}\\record_index.json",
  "event_candidates_path": "{data_root}\\metadata\\{match_id}\\event_candidates.json",
  "clip_policy": {
    "goal_candidate": { "pre_sec": 8, "post_sec": 10 },
    "shot_candidate": { "pre_sec": 6, "post_sec": 6 },
    "danger_attack_candidate": { "pre_sec": 5, "post_sec": 5 },
    "celebration_candidate": { "pre_sec": 3, "post_sec": 8 }
  },
  "max_duration_sec": 600,
  "max_event_count": 15
}
```

响应：
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "match_id": "match_20260405_001",
    "task_id": "hl_task_20260405_001",
    "output_path": "{data_root}\\output\\match_20260405_001\\full_highlight.mp4",
    "status": "processing",
    "estimated_sec": 180
  }
}
```

### 9.2 状态查询

**方法**：`GET`
**路径**：`/api/v1/highlight/tasks/{task_id}/status`

响应：
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "task_id": "hl_task_20260405_001",
    "status": "success",
    "output_path": "{data_root}\\output\\match_20260405_001\\full_highlight.mp4",
    "duration_sec": 485,
    "event_count": 8,
    "events_used": ["evt_001", "evt_002", "evt_003"],
    "events_skipped": ["evt_004"]
  }
}
```

## 10. 错误处理

| 错误场景 | 错误码 | 处理 |
|------|------|------|
| 录像索引文件不存在 | 1006 | 返回文件不存在错误 |
| 事件文件为空 | 0 | 返回空集锦路径（无事件，不生成集锦）|
| 片段裁切失败 | 1007 | 跳过该片段，继续处理其余片段 |
| 拼接失败 | 1007 | 返回完整错误日志 |
| 磁盘空间不足 | 1019 | 终止生成，返回空间不足告警 |
