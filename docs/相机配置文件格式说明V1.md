## 1. 配置文件路径

项目根目录下，与 `services/` 并列：

```
configs/ingest_streaming/config.json
```

程序内部按以下顺序查找配置文件（第一个找到的即使用）：
1. `{exe所在目录}/config.json`
2. `{exe所在目录}/configs/ingest_streaming/config.json`
3. `{exe所在目录}/../../../configs/ingest_streaming/config.json`（VS编译输出目录）
4. `{exe所在目录}/../../../../configs/ingest_streaming/config.json`（更深VS输出目录）
5. `configs/ingest_streaming/config.json`（开发时相对路径）
6. 绝对路径（代码中兜底）

## 2. 根结构

```json
{
  "data_root": "./data",
  "log_level": "info",
  "cameras": []
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `data_root` | string | 数据根目录（当前未使用，保留字段） |
| `log_level` | string | 日志级别：`debug`/`info`/`warn`/`error` |
| `cameras` | array | 相机配置数组 |

## 3. 单相机配置结构（冻结字段）

```json
{
  "serial": "从相机外壳或MVS客户端获取的序列号",
  "camera_id": "cam_01",
  "role": "main",
  "width": "2592",
  "height": "1944",
  "fps": "25.0",
  "rtsp_url": "rtsp://127.0.0.1:8554/main"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `serial` | string | 相机唯一序列号，用于SDK枚举定位 |
| `camera_id` | string | 逻辑ID，**必须为 `cam_01` 或 `cam_02`**（合同约定） |
| `role` | string | 角色：`main`（主机位）或 `aux`（辅机位） |
| `width` | string/int | 相机分辨率宽，当前实机为 2592 |
| `height` | string/int | 相机分辨率高，当前实机为 1944 |
| `fps` | string/double | 帧率，当前实机为 25.0 |
| `rtsp_url` | string | RTSP推流目标地址，**必须与MediaMTX实例端口和路径匹配** |

> **分辨率说明**：实机相机为海康威视 GigE 相机，原生分辨率 2592x1944（4:3）。程序内部保持原生分辨率编码输出，不再缩放至 16:9，避免画面变形。

## 5. 相机序列号获取方法

1. 查看相机外壳标签（序列号印在设备铭牌上）
2. 通过 MVS 客户端（海康威视 MVS 软件）连接相机后查看
3. 运行相机枚举程序枚举所有已连接 GigE/USB 相机，输出序列号

## 6. 常见配置错误

| 错误 | 症状 | 解决方法 |
|------|------|----------|
| `serial` 与实际相机不匹配 | 相机初始化失败，日志显示"Camera X not found" | 核对 MVS 客户端中的相机序列号 |
| `rtsp_url` 端口与 MediaMTX 配置不一致 | RTSP 推流失败，日志显示连接错误 | 确认 MediaMTX 配置文件中端口与 RTSP URL 匹配 |
| `camera_id` 非 `cam_0x` 格式 | HTTP 接口返回ID不符合合同约定 | 必须使用 `cam_01`/`cam_02` |
| 相机分辨率字符串含空格 | 参数解析异常，设为默认值 2592x1944 | 确保 JSON 中 `width`/`height` 为纯数字字符串 |
