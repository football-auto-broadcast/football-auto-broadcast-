# 模块联调说明 v1

## 文档信息
- 版本：v1
- 适用阶段：第 3-5 周核心联调
- 适用团队：5 名工程师（A/B/C/D/E）
- 测试环境：Windows 10/11 单主机，RTX 4080

---

## 1. 联调前提

在开始联调之前，确保：
1. 所有模块均已完成第 2 周"单模块最小可运行"验收
2. GigE 工业相机已通过 MVS 客户端验证枚举成功
3. 项目 `configs/` 目录下所有配置文件已就绪
4. 各模块 HTTP 端口无冲突（8080-8084）
5. 磁盘剩余空间 ≥ 50GB

---

## 2. 联调顺序与步骤

### 第一步：A 模块单独联通（采集模块）

**目标**：
1. 两台工业相机可被识别
2. SDK 初始化成功
3. 两路视频可持续获取
4. 主机位和辅机位内部流可播放（`rtsp://127.0.0.1:8554/main`、`rtsp://127.0.0.1:8555/aux`）

**操作步骤**：
```bat
cd services\ingest-streaming-module\bin
.\ingest-streaming-module.exe
```

**验证方法**：
- 控制台输出显示两路相机的分辨率、帧率、在线状态
- `ffplay -rtsp_transport tcp rtsp://127.0.0.1:8554/main` 可见 cam_01 画面
- `ffplay -rtsp_transport tcp rtsp://127.0.0.1:8555/aux` 可见 cam_02 画面
- 浏览器访问 `http://127.0.0.1:8081/api/v1/ingest/status` 返回正常 JSON
- 连续运行 30 分钟无崩溃

**验收人**：工程师 A

---

### 第二步：A + B 联调（采集 → 录像）

**目标**：
1. B 模块从 A 模块的 RTSP 输入获取两路视频流
2. B 模块能按比赛维度创建录像目录
3. 双路原始录像可保存到 `{data_root}\raw\{match_id}\`
4. cam_01/cam_02 默认裁切画面可生成
5. `record_index.json` 可生成

**操作步骤**：
1. 确保 A 模块正在运行（两路 RTSP 可用）
2. 启动 B 模块：`record_program_service.exe`
3. 通过平台 API 创建比赛：
   ```bat
   curl -X POST http://127.0.0.1:8082/api/v1/record/matches/init -H "Content-Type: application/json" -d @init_record.json
   ```
4. 通过平台 API 开始录制：
   ```bat
   curl -X POST http://127.0.0.1:8082/api/v1/record/matches/match_test_001/start
   ```
5. 运行 5 分钟后停止：
   ```bat
   curl -X POST http://127.0.0.1:8082/api/v1/record/matches/match_test_001/stop
   ```

**验证方法**：
- `{data_root}\raw\match_test_001\` 下存在 `cam_01.mp4` 和 `cam_02.mp4`
- `{data_root}\program\match_test_001\` 下存在 `cam_01_cut.mp4`、`cam_02_cut.mp4`、`program.mp4`
- `{data_root}\metadata\match_test_001\record_index.json` 内容正确
- 录制文件可正常播放

**验收人**：工程师 A + B

---

### 第三步：A + C 联调（采集 → 视觉）

**目标**：
1. C 模块从 A 模块的 RTSP 输入获取两路视频流
2. C 模块可输出双机位关注区域
3. C 模块可输出候选事件（至少 goal_candidate 和 shot_candidate）
4. C 模块可输出多机位决策结果

**操作步骤**：
1. 确保 A 模块正在运行
2. 启动 C 模块：`vision_event_service.exe`
3. 通过 API 初始化：
   ```bat
   curl -X POST http://127.0.0.1:8083/api/v1/vision/matches/init -H "Content-Type: application/json" -d @init_vision.json
   ```
4. 运行 5 分钟后停止

**验证方法**：
- C 模块日志中持续输出双机位关注区域
- `event_candidates.json` 不为空（至少包含 1 个候选事件）
- 候选事件时间窗和置信度合理
- 关注区域每 200ms 输出一次

**验收人**：工程师 A + C

---

### 第四步：B + C 联调（视觉 → 主画面）

**目标**：
1. B 模块消费 C 模块输出的双机位关注区域
2. B 模块根据关注区域动态裁切主画面
3. B 模块根据多机位决策切换画面来源
4. program_decision 超时时 B 正确回退 cam_01

**操作步骤**：
1. 确保 A、B、C 模块都在运行
2. 触发 B 模块初始化并与 C 模块关联
3. 观察主画面预览 `rtsp://127.0.0.1:8560/program`
4. 模拟 C 模块停止推送 → 验证 B 回退

**验证方法**：
- 主画面预览可见动态裁切（画面随关注区域移动）
- 手动停止 C 模块的 program_decision 推送 → B 应回退 cam_01
- 画面切换平滑（无明显抖动）

**验收人**：工程师 B + C

---

### 第五步：C + D 联调（事件 → 集锦）

**目标**：
1. D 模块读取 C 模块输出的事件文件
2. D 模块读取 B 模块输出的录像索引
3. D 模块生成全场精彩集锦

**操作步骤**：
1. 确保 A、B、C 完成过至少一场测试（已有录像和事件文件）
2. 启动 D 模块：`highlight_generation_service.exe`
3. 触发集锦生成：
   ```bat
   curl -X POST http://127.0.0.1:8084/api/v1/highlight/matches/match_test_001/generate -H "Content-Type: application/json" -d @gen_highlight.json
   ```

**验证方法**：
- `{data_root}\output\match_test_001\full_highlight.mp4` 可正常播放
- 集锦包含至少一个候选事件片段
- 片段拼接顺序符合优先级规则

**验收人**：工程师 C + D

---

### 第六步：E 接入所有模块

**目标**：
1. 后台可创建比赛 → 开始 → 结束
2. 后台可查看各模块状态
3. 后台可预览主画面
4. 后台可触发集锦生成并下载结果

**操作步骤**：
1. 确保 A、B、C、D 模块全部启动
2. 启动 E 模块：`platform_orchestration_service.exe`
3. 浏览器访问 `http://127.0.0.1:8080`
4. 按照"创建 → 开始 → 预览 → 结束 → 集锦生成 → 下载"流程操作

**验证方法**：
- 比赛控制页可完成创建/开始/结束
- 状态页显示各模块状态、双机位信息、录制时长
- 主画面预览页可看到实时画面
- 集锦结果页可下载 `full_highlight.mp4`

**验收人**：全团队

## 3. 联调检查清单

| 检查项 | 要求 | 通过 |
|------|------|------|
| 双路相机取流 | 两路 RTSP 均可用 | ☐ |
| 双路原始录像 | cam_01.mp4 + cam_02.mp4 可写 | ☐ |
| 裁切画面输出 | cam_01_cut + cam_02_cut 可生成 | ☐ |
| 主画面预览 | rtsp://127.0.0.1:8560/program 可播放 | ☐ |
| 双机位关注区域 | 200ms 一次稳定输出 | ☐ |
| 候选事件输出 | event_candidates.json 非空 | ☐ |
| 多机位决策输出 | program_decision 按需推送 | ☐ |
| record_index.json | 索引结构正确 | ☐ |
| 动态裁切联动 | 主画面随关注区域移动 | ☐ |
| 决策超时回退 | program_decision 超时回退 cam_01 | ☐ |
| 集锦生成 | full_highlight.mp4 可正常播放 | ☐ |
| 平台全流程 | 创建→开始→结束→集锦→下载 | ☐ |

## 4. 常见联调问题

| 问题 | 原因 | 解决 |
|------|------|------|
| RTSP 端口不通 | MediaMTX 未启动 | 确保 ingest-streaming-module 已启动 |
| 录像文件无法创建 | 目录无写权限 | 检查 data_root 目录权限 |
| 关注区域未推送 | C 模块断连 | 检查 C 模块日志 |
| program_decision 始终不变 | 视觉推理未检测到变化 | 尝试移动相机前物体 |
| 集锦片段长度异常 | 事件时间和录像时间不对齐 | 统一使用相对比赛时间 start_sec |
| ffplay 花屏 | 编码 qp-max 过大 | 见 ingest 配置 bitrate 和 qp-max 参数 |
