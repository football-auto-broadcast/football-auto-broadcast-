# record-program-module
## 模块名称
录像与主画面生成模块

## 英文名
`record-program-module`

## 模块定位
本模块负责两路比赛录像保存、双机位裁切画面生成、主画面选择与输出、主画面录制与文件归档。  
该模块直接面向比赛过程中的“录什么、裁什么、播什么”。

在当前架构下，本模块运行于 RTX 4080 主机本地，接收采集模块提供的两路视频输入，接收视觉模块输出的双机位关注区域与多机位决策结果，并生成：

1. `cam_01` 裁切画面  
2. `cam_02` 裁切画面  
3. 最终主画面  
4. 原始录像与主画面录像

## 主流程说明
### 1. 原始录像
本模块必须分别录制：

+ `cam_01` 原始视频
+ `cam_02` 原始视频

### 2. 双机位裁切
本模块必须分别对两路视频执行裁切：

+ 根据 `cam_01` 关注区域生成 `cam_01` 裁切画面
+ 根据 `cam_02` 关注区域生成 `cam_02` 裁切画面

### 3. 主画面选择
本模块根据视觉模块输出的 `program_decision` 选择当前主画面来源机位。

### 4. 主画面输出
本模块输出一路最终主画面，供：

+ 平台预览
+ 主画面录像
+ 后续集锦优先参考

---

## 多机位画面策略
### 策略 1：双机位都必须可用
当前版本的硬要求是：

1. `cam_01` 裁切画面必须可生成  
2. `cam_02` 裁切画面必须可生成  
3. 不允许只有主机位能裁切、辅机位不能用

### 策略 2：视觉决策优先
如果视觉模块输出了 `recommended_camera_id`，本模块优先使用对应机位作为当前主画面来源。

### 策略 3：默认回退
当视觉决策缺失、异常或超时，本模块必须回退到默认主机位策略：

+ 默认主画面机位为 `cam_01`

### 策略 4：单路异常隔离
如果某一路关注区域无效：

1. 只回退该路默认区域  
2. 不影响另一机位继续工作  
3. 不允许整条主画面链路直接中断

## 任务清单
### 1. 原始录像
1. 对主机位和辅机位分别录制原始视频  
2. 支持按比赛维度创建目录  
3. 支持文件命名规范化  
4. 支持比赛开始与结束控制  
5. 保证文件可被后续集锦模块读取

### 2. 双机位裁切
1. 接收 `cam_01` 关注区域  
2. 接收 `cam_02` 关注区域  
3. 分别生成两路裁切画面  
4. 视觉信号缺失时分别回退默认区域  
5. 支持两路裁切平滑处理

### 3. 主画面选择
1. 接收视觉模块输出的多机位决策结果  
2. 支持根据推荐机位选择当前主画面  
3. 支持默认主机位回退  
4. 支持决策结果超时保护  
5. 支持当前主画面机位状态查询

### 4. 主画面输出
1. 输出一路可预览的最终主画面  
2. 支持主画面录制  
3. 支持后续平台网页预览接入  
4. 支持本地预览或标准化流输出

### 5. 文件归档
1. 保存原始录像文件  
2. 保存主画面录像文件  
3. 可选保存双机位裁切录像  
4. 记录文件路径、所属比赛、起止时间等信息  
5. 向集锦模块提供标准化文件访问方式

### 6. 与其他模块协作
1. 接收 `ingest-streaming-module` 提供的视频输入  
2. 接收 `vision-event-module` 输出的双机位关注区域  
3. 接收 `vision-event-module` 输出的多机位决策结果  
4. 向 `highlight-generation-module` 提供录像文件  
5. 向 `platform-orchestration-module` 提供录制状态和文件索引

## 运行原则
当前版本中，本模块不自行做复杂视觉判断。  
本模块只做三件事：

1.  根据双机位关注区域裁切 
2.  根据多机位决策结果选择主画面 
3.  负责录制、输出与归档 

这样可以保证模块边界清晰，便于与视觉模块、平台模块联调。
