# 模块 D：事件与集锦生成模块

## 模块名称
事件与集锦生成模块

## 英文名
`highlight-generation-module`

## 对应工程师
工程师 D

## 模块定位
本模块负责将视觉模块输出的候选事件和录像文件转化为最终的全场精彩集锦。  
该模块聚焦“进球”和“精彩片段”两类内容，不再承担个人集锦生成职责。

---

## MVP Pro 目标

1. 读取比赛录像文件和视觉事件数据。
2. 根据候选事件裁切视频片段。
3. 生成全场精彩集锦。
4. 支持事件筛选、片段排序和导出。
5. 为平台模块提供集锦结果和任务状态。

---

## 任务清单

### 1. 事件接收与解析
1. 接收视觉分析模块输出的候选事件。
2. 支持解析事件类型、时间窗、置信度。
3. 对事件进行基础去重和筛选。
4. 对事件按优先级进行组织。

### 2. 视频片段裁切
1. 从录像文件中读取对应时间段视频。
2. 对事件前后增加缓冲时间。
3. 支持按主画面录像优先裁切。
4. 在必要时支持从原始录像中补充裁切。

### 3. 全场精彩集锦生成
1. 根据候选事件拼接精彩片段。
2. 支持进球片段优先进入集锦。
3. 支持控制集锦总时长。
4. 支持导出标准视频文件。

### 4. 精彩片段排序策略
1. 进球优先级最高。
2. 射门、危险进攻、庆祝次之。
3. 支持按置信度和时间顺序排序。
4. 支持最简规则配置化。

### 5. 导出与结果上报
1. 导出全场集锦文件。
2. 返回集锦文件路径。
3. 上报任务执行状态。
4. 输出失败日志与错误信息。

### 6. 与其他模块协作
1. 从录像与主画面生成模块读取视频文件。
2. 从视觉分析模块读取事件数据。
3. 向平台模块上报任务进度与结果。

---

## 成功验收标准

1. 比赛结束后能生成一条全场精彩集锦。
2. 集锦中至少包含进球和部分精彩候选片段。
3. 集锦文件能正常播放和导出。
4. 平台模块可查询任务状态和输出路径。
5. 失败时能输出明确错误日志。
6. 模块可独立以 Docker 方式部署运行。
