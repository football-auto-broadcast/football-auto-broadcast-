# highlight-generation-module
## 模块名称
事件与集锦生成模块

## 英文名
`highlight-generation-module`

## 模块定位
本模块负责将视觉模块输出的候选事件和录像文件转化为最终的全场精彩集锦。  
该模块聚焦“进球”和“精彩片段”两类内容，不承担个人集锦生成职责。

在新架构下，本模块运行在同一台 RTX 4080 主机上，直接读取本地录像文件和本地候选事件数据，不再依赖分布式节点之间的文件同步。

---

## 所在架构位置
本模块接收：

+ `record-program-module` 提供的录像文件
+ `vision-event-module` 输出的事件候选数据

并输出：

+ 全场精彩集锦视频
+ 任务状态
+ 导出结果信息

---

## MVP Pro 目标
1. 读取比赛录像文件和视觉事件数据。
2. 根据候选事件裁切视频片段。
3. 生成全场精彩集锦。
4. 支持事件筛选、片段排序和导出。
5. 为平台模块提供集锦结果和任务状态。

---

## 任务清单
### 1. 事件接收与解析
1. 接收视觉分析模块输出的候选事件。
2. 支持解析事件类型、时间窗、置信度。
3. 对事件进行基础去重和筛选。
4. 对事件按优先级进行组织。

### 2. 视频片段裁切
1. 从录像文件中读取对应时间段视频。
2. 对事件前后增加缓冲时间。
3. 支持按主画面录像优先裁切。
4. 在必要时支持从原始录像中补充裁切。

### 3. 全场精彩集锦生成
1. 根据候选事件拼接精彩片段。
2. 支持进球片段优先进入集锦。
3. 支持控制集锦总时长。
4. 支持导出标准视频文件。

### 4. 精彩片段排序策略
1. 进球优先级最高。
2. 射门、危险进攻、庆祝次之。
3. 支持按置信度和时间顺序排序。
4. 支持最简规则配置化。

### 5. 导出与结果上报
1. 导出全场精彩集锦文件。
2. 返回集锦文件路径。
3. 上报任务执行状态。
4. 输出失败日志与错误信息。

### 6. 与其他模块协作
1. 从 `record-program-module` 读取录像文件。
2. 从 `vision-event-module` 读取事件数据。
3. 向 `platform-orchestration-module` 上报任务进度与结果。

---

## 输入
1. 原始录像文件
2. 主画面录像文件
3. 候选事件数据
4. 集锦导出配置

---

## 输出
1. 全场精彩集锦文件
2. 集锦文件路径
3. 任务状态
4. 事件使用统计
5. 错误日志

---

## 成功验收标准
1. 比赛结束后能生成一条全场精彩集锦。
2. 集锦中至少包含进球和部分精彩候选片段。
3. 集锦文件能正常播放和导出。
4. 平台模块可查询任务状态和输出路径。
5. 失败时能输出明确错误日志。
6. 在单主机集中式架构下可稳定运行。
