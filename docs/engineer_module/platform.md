# 模块 E：平台与调度模块

## 模块名称
平台与调度模块

## 英文名
`platform-orchestration-module`

## 对应工程师
工程师 E

## 模块定位
本模块负责整套系统的管理后台、任务调度、服务状态聚合、配置管理和运维协作。  
该模块是操作入口，也是各服务之间的统一控制中心。

本模块可提供一个内部使用的网页端，用于比赛控制、状态查看和精彩集锦结果展示。

---

## MVP Pro 目标

1. 提供比赛管理入口。
2. 提供各服务统一调度能力。
3. 提供状态查看与日志汇总能力。
4. 提供主画面预览页面和结果查看页面。
5. 统一 Docker 启动方式和 Git 协作规范。

---

## 任务清单

### 1. 比赛管理
1. 支持创建比赛。
2. 支持开始比赛。
3. 支持结束比赛。
4. 支持查看比赛列表和比赛状态。

### 2. 服务调度
1. 调用采集与编码分发模块进行初始化和启动。
2. 调用录像与主画面生成模块进行录制控制。
3. 调用视觉分析模块进行启动和停止。
4. 调用集锦模块进行全场精彩集锦生成。
5. 统一管理任务状态。

### 3. 后台页面
1. 提供比赛控制页面。
2. 提供服务状态页面。
3. 提供主画面预览页面。
4. 提供集锦结果查看和下载页面。
5. 保持页面功能简洁，满足内部操作即可。

### 4. 配置管理
1. 管理相机配置。
2. 管理服务地址配置。
3. 管理输出目录配置。
4. 管理基础事件规则配置。
5. 支持多节点 IP 和端口管理。

### 5. 日志与状态汇总
1. 汇总各模块运行状态。
2. 汇总关键错误日志。
3. 在后台显示模块在线状态。
4. 支持任务执行状态查询。

### 6. 工程与运维支持
1. 维护 Docker Compose 或分布式部署说明。
2. 维护启动文档和运行文档。
3. 维护 Git 分支协作规范。
4. 为联调提供统一配置和测试入口。

---

## 成功验收标准

1. 可通过后台完成创建比赛、开始比赛、结束比赛。
2. 可查看各模块在线状态和任务状态。
3. 可预览自动转播主画面。
4. 可触发全场精彩集锦生成并查看结果。
5. 可下载生成后的集锦文件。
6. 项目具备基础部署文档和协作文档。
7. 模块可独立以 Docker 方式部署运行。
