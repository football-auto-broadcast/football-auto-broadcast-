# platform-orchestration-module
## 模块名称
平台与调度模块

## 英文名
`platform-orchestration-module`

## 模块定位
本模块负责整套系统的管理后台、任务调度、服务状态聚合、配置管理和运维协作。  
该模块是操作入口，也是各逻辑模块之间的统一控制中心。

在新架构下，本模块与其他模块统一部署在同一台主机上，但仍然保留清晰的逻辑边界。  
本模块同时可向外提供一个内部使用的网页端，用于比赛控制、状态查看和精彩集锦结果展示。

---

## 所在架构位置
本模块直接调度：

+ `ingest-streaming-module`
+ `record-program-module`
+ `vision-event-module`
+ `highlight-generation-module`

同时向可选控制端提供：

+ 比赛管理页面
+ 主画面预览页面
+ 集锦结果查看与下载页面

---

## MVP Pro 目标
1. 提供比赛管理入口。
2. 提供各模块统一调度能力。
3. 提供状态查看与日志汇总能力。
4. 提供主画面预览页面和结果查看页面。
5. 提供统一配置入口和工程协作支撑。

---

## 任务清单
### 1. 比赛管理
1. 支持创建比赛。
2. 支持开始比赛。
3. 支持结束比赛。
4. 支持查看比赛列表和比赛状态。

### 2. 服务调度
1. 调用采集模块进行初始化和启动。
2. 调用录像与主画面模块进行录制控制。
3. 调用视觉分析模块进行启动和停止。
4. 调用集锦模块进行全场精彩集锦生成。
5. 统一管理任务状态。

### 3. 后台页面
1. 提供比赛控制页面。
2. 提供服务状态页面。
3. 提供主画面预览页面。
4. 提供集锦结果查看和下载页面。
5. 保持页面功能简洁，满足内部操作即可。

### 4. 配置管理
1. 管理工业相机参数配置。
2. 管理模块配置。
3. 管理输出目录配置。
4. 管理基础事件规则配置。
5. 支持主机内部统一服务配置加载。

### 5. 日志与状态汇总
1. 汇总各模块运行状态。
2. 汇总关键错误日志。
3. 在后台显示模块在线状态。
4. 支持任务执行状态查询。

### 6. 工程与运维支持
1. 维护单主机部署说明。
2. 维护启动文档和运行文档。
3. 维护 Git 分支协作规范。
4. 为联调提供统一配置和测试入口。

---

## 输入
1. 比赛控制请求
2. 各模块状态信息
3. 配置文件
4. 集锦任务结果

---

## 输出
1. 比赛管理页面
2. 主画面预览页面
3. 状态页面
4. 集锦查看与下载页面
5. 调度命令
6. 运行日志摘要

---

## 成功验收标准
1. 可通过后台完成创建比赛、开始比赛、结束比赛。
2. 可查看各模块在线状态和任务状态。
3. 可预览自动转播主画面。
4. 可触发全场精彩集锦生成并查看结果。
5. 可下载生成后的集锦文件。
6. 项目具备基础部署文档和协作文档。
7. 在单主机集中式架构下可稳定运行。
