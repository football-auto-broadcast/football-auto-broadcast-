# record-program-module
## 模块名称
录像与主画面生成模块

## 英文名
`record-program-module`

## 模块定位
本模块负责两路比赛录像保存、自动转播主画面生成、主画面录制与文件归档。  
该模块直接面向比赛过程中的“录什么”和“播什么”。

在新架构下，本模块与采集模块同处于同一台主机，不再依赖跨节点视频拉流，而是直接消费主机内部统一视频输入。

---

## 所在架构位置
本模块位于 RTX 4080 主机内部，接收：

+ `ingest-streaming-module` 的主机位和辅机位视频输入
+ `vision-event-module` 输出的关注区域或裁切建议

并输出：

+ 原始录像文件
+ 主画面录像文件
+ 主画面预览输出
+ 文件索引与状态信息

---

## MVP Pro 目标
1. 录制两路比赛原始视频。
2. 基于主机位生成一路基础自动转播主画面。
3. 对主画面进行平滑裁切与输出。
4. 保存原始录像和主画面录像文件。
5. 为赛后集锦模块提供标准化视频文件索引。

---

## 任务清单
### 1. 原始录像
1. 对主机位和辅机位分别录制原始视频。
2. 支持按比赛维度创建目录。
3. 支持文件命名规范化。
4. 支持比赛开始与结束控制。
5. 保证文件可被后续集锦模块读取。

### 2. 主画面生成
1. 从主机位视频中生成一路自动转播主画面。
2. 接收视觉模块输出的关注区域。
3. 根据关注区域执行裁切。
4. 在视觉信号缺失时回退到默认中心区域。
5. 优先保证主画面连续可看，而不是追求复杂切镜。

### 3. 画面平滑
1. 对裁切窗口执行平滑过渡。
2. 减少画面快速跳动。
3. 避免出现剧烈裁切抖动。
4. 支持基础参数配置，例如平滑系数和最大移动步长。

### 4. 主画面输出
1. 输出一路可预览的自动转播主画面。
2. 支持主画面录制。
3. 支持后续平台网页预览接入。
4. 支持本地预览或标准化流输出。

### 5. 文件归档
1. 保存原始录像文件。
2. 保存主画面录像文件。
3. 记录文件路径、所属比赛、起止时间等信息。
4. 向集锦模块提供标准化文件访问方式。

### 6. 与其他模块协作
1. 接收 `ingest-streaming-module` 提供的视频输入。
2. 接收 `vision-event-module` 输出的关注区域。
3. 向 `highlight-generation-module` 提供录像文件。
4. 向 `platform-orchestration-module` 提供录制状态和文件索引。

---

## 输入
1. 主机位视频输入
2. 辅机位视频输入
3. 关注区域
4. 比赛开始与结束命令
5. 文件输出配置

---

## 输出
1. 原始录像文件
2. 主画面录像文件
3. 主画面预览输出
4. 文件索引信息
5. 模块状态与日志

---

## 成功验收标准
1. 能完成两路比赛原始视频录制。
2. 能生成一路基础自动转播主画面。
3. 主画面裁切具有基础平滑效果。
4. 比赛结束后可找到完整录像文件和主画面文件。
5. 集锦模块可读取对应录像文件。
6. 平台模块可查询录制状态和文件路径。
7. 在单主机集中式架构下可稳定运行。
