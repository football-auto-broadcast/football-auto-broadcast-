# 模块 B：录像与主画面生成模块

## 模块名称
录像与主画面生成模块

## 英文名
`record-program-module`

## 对应工程师
工程师 B

## 模块定位
本模块负责比赛录像保存、自动转播主画面生成、主画面录制与文件归档。  
该模块直接面向比赛过程中的“录什么”和“播什么”。

本模块部署在采集节点，与采集与编码分发模块协作最紧密。

---

## MVP Pro 目标

1. 录制两路比赛原始视频。
2. 基于主机位生成一路基础自动转播主画面。
3. 对主画面进行平滑裁切与输出。
4. 保存原始录像和主画面录像文件。
5. 为赛后集锦模块提供可读取的视频文件索引。

---

## 任务清单

### 1. 原始录像
1. 对主机位和辅机位分别录制原始视频。
2. 支持按比赛维度创建存储目录。
3. 支持文件命名规范化。
4. 支持比赛开始和结束控制。

### 2. 主画面生成
1. 从主机位视频中生成一路自动转播主画面。
2. 接收视觉模块提供的关注区域或裁切建议。
3. 根据建议执行裁切。
4. 在视觉信号不稳定时支持回退到默认居中策略。

### 3. 画面平滑
1. 对裁切窗口执行平滑过渡。
2. 减少画面快速跳动。
3. 避免出现明显抖动和剧烈切换。
4. 支持基础参数配置，例如平滑系数和最大移动速度。

### 4. 主画面输出
1. 输出一路可预览的自动转播主画面。
2. 支持主画面录制。
3. 支持后续平台网页预览接入。
4. 支持本地播放或网络分发。

### 5. 文件归档
1. 保存原始录像文件。
2. 保存主画面录像文件。
3. 记录文件路径、起止时间、所属比赛等信息。
4. 为集锦模块提供标准化文件访问方式。

### 6. 与其他模块协作
1. 接收采集与编码分发模块的视频输入。
2. 接收视觉分析模块的关注区域输出。
3. 向集锦模块提供录像文件。
4. 向平台模块提供录制状态和文件索引。

---

## 成功验收标准

1. 能完成两路比赛原始视频录制。
2. 能生成一路基础自动转播主画面。
3. 主画面裁切具有基础平滑效果。
4. 比赛结束后可找到完整录像文件和主画面文件。
5. 平台模块可查询录制状态和文件路径。
6. 模块可独立以 Docker 方式部署运行。
