# vision-event-module
## 模块名称
视觉分析模块

## 英文名
`vision-event-module`

## 模块定位
本模块负责从比赛视频中提取与自动转播和精彩集锦相关的基础视觉信号。  
本模块不负责球员号码识别、球员身份区分和个人集锦支持，只关注比赛中的关键区域、进球候选和精彩片段候选。

在新架构下，本模块运行于 RTX 4080 主机本地，直接消费采集模块输出的视频输入，不再依赖跨节点视频拉流。

---

## 所在架构位置
本模块位于单主机集中式系统内部，接收：

+ `ingest-streaming-module` 提供的视频输入

并输出：

+ 给 `record-program-module` 的关注区域
+ 给 `highlight-generation-module` 的候选事件数据

---

## MVP Pro 目标
1. 提供主画面裁切所需的关注区域信息。
2. 识别进球候选事件。
3. 识别精彩片段候选事件。
4. 输出可供集锦模块直接使用的结构化事件线索。
5. 为系统提供轻量、稳定、可落地的视觉分析能力。

---

## 任务清单
### 1. 关注区域检测
1. 分析广角比赛画面中的主要活动区域。
2. 输出当前推荐关注区域。
3. 为主画面生成模块提供裁切建议。
4. 在球检测不稳定时，支持使用运动聚集区域作为替代信号。

### 2. 足球基础检测
1. 尽量检测足球在画面中的大致位置。
2. 输出球位置及其置信度。
3. 将球位置信息用于主画面关注区域估计。
4. 在球无法持续检测时允许短时丢失。

### 3. 高活跃区域分析
1. 分析球门附近、禁区附近和进攻方向的运动强度。
2. 判断门前聚集、快速反击、明显攻势。
3. 输出高活跃区域时间窗。
4. 作为精彩片段候选的重要依据。

### 4. 进球候选识别
1. 检测球接近球门或球门区域剧烈变化。
2. 结合赛场画面节奏变化判断进球候选。
3. 输出 `goal_candidate` 类型事件。
4. 记录候选事件时间窗和置信度。

### 5. 精彩片段候选识别
1. 识别明显射门候选。
2. 识别危险进攻候选。
3. 识别庆祝片段候选。
4. 输出事件起止时间和置信度。
5. 统一生成可供集锦模块消费的事件数据。

### 6. 元数据输出
1. 输出统一 JSON 结构。
2. 为每个事件携带时间信息。
3. 为每个事件携带事件类型和置信度。
4. 向录像模块输出关注区域。
5. 向集锦模块输出候选事件。

---

## 输入
1. 主机位视频输入
2. 辅机位视频输入
3. 视觉参数配置
4. 事件规则配置

---

## 输出
1. 关注区域
2. 进球候选事件
3. 精彩片段候选事件
4. 候选事件 JSON 数据
5. 模块状态与日志

---

## 成功验收标准
1. 能输出主画面裁切所需的关注区域。
2. 能输出进球候选事件。
3. 能输出精彩片段候选事件。
4. 输出的事件结构可被集锦模块直接消费。
5. 在广角工业相机场景下保持基础可用性。
6. 在单主机集中式架构下可稳定运行。
