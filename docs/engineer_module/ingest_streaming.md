# 模块 A：采集与编码分发模块

## 模块名称
采集与编码分发模块

## 英文名
`ingest-streaming-module`

## 对应工程师
工程师 A

## 模块定位
本模块负责将两路相机视频接入系统，并在采集节点完成统一编码、推流与基础流状态管理，为后续视觉分析模块和集锦模块提供可消费的视频流输入。

本模块是整套系统的视频入口，也是局域网视频分发中心。

---

## MVP Pro 目标

1. 接入两路比赛相机视频。
2. 在采集节点完成统一视频编码压缩。
3. 通过局域网向其他节点稳定分发视频流。
4. 提供视频流状态监控与异常恢复能力。
5. 为录像模块和视觉模块提供标准化视频输入源。

---

## 任务清单

### 1. 相机接入
1. 支持接入两路相机视频。
2. 支持 RTSP、USB 采集或 HDMI 采集卡输入。
3. 支持通过配置文件管理相机参数。
4. 支持区分主机位和辅机位。

### 2. 视频编码
1. 在采集节点对两路视频执行统一编码压缩。
2. 优先支持 H.265 硬件编码。
3. 在硬件条件不足时支持 H.264 或软件编码降级。
4. 支持配置分辨率、码率、帧率。

### 3. 视频流分发
1. 向局域网发布标准视频流。
2. 推荐提供 RTSP 流地址。
3. 为视觉模块提供主机位和辅机位视频流。
4. 为后续扩展保留更多订阅端的接入能力。

### 4. 流状态监控
1. 监控每路流的在线状态。
2. 监控实际分辨率、帧率、码率。
3. 视频输入断开时输出错误日志。
4. 在可行范围内支持自动重连。

### 5. 配置与接口
1. 提供统一初始化接口。
2. 提供启动和停止接口。
3. 提供流地址查询接口。
4. 提供状态查询接口。
5. 与平台模块保持配置一致。

### 6. 与其他模块协作
1. 向录像与主画面模块提供采集后的视频源。
2. 向视觉分析模块提供视频流地址。
3. 向平台模块上报状态和异常。

---

## 成功验收标准

1. 能稳定接入两路相机视频。
2. 能输出可被其他节点拉取的标准视频流。
3. 两路视频流在局域网中可持续访问。
4. 平台模块可查询到每路流状态。
5. 相机断开或异常时有清晰日志输出。
6. 模块可独立以 Docker 方式部署运行。
