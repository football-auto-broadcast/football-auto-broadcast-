# ingest-streaming-module
## 模块名称
采集与编码分发模块

## 英文名
`ingest-streaming-module`

## 模块定位
本模块负责将两路 GigE 工业相机接入系统，并在主机上完成统一取流、基础预处理、编码压缩与流分发。

在新架构中，本模块不再面向“多台 PC 分布式采集节点”，而是作为**单主机集中式系统的视频入口**存在。  
本模块直接对接工业相机 SDK，是整套系统最底层、最关键的视频输入模块。

---

## 所在架构位置
当前系统采用以下硬件链路：

工业相机 1 + 工业相机 2  
→ 千兆 PoE 交换机  
→ RTX 4080 主机  
→ 本模块完成工业相机 SDK 取流与编码分发

本模块输出给：

+ `record-program-module`
+ `vision-event-module`

---

## MVP Pro 目标
1. 通过工业相机 SDK 稳定接入两路 GigE 工业相机。
2. 完成两路视频流的基础管理与状态监控。
3. 根据系统需要提供标准化视频输入源。
4. 支持本机内部的视频流分发或共享帧输入。
5. 为录像模块与视觉模块提供可持续、稳定的视频源。

---

## 任务清单
### 1. 工业相机接入
1. 支持加载厂商工业相机 SDK。
2. 支持枚举并识别两台 GigE 工业相机。
3. 支持按配置文件绑定主机位与辅机位。
4. 支持读取和设置相机核心参数，例如曝光、增益、分辨率、帧率。
5. 支持相机初始化失败时输出明确日志。

### 2. 视频取流
1. 从两台工业相机持续获取图像帧。
2. 支持连续取流与异常断流检测。
3. 支持在相机掉线后进行错误上报。
4. 支持为上层模块提供统一帧数据接口或标准内部流接口。

### 3. 编码与分发
1. 根据系统配置完成基础视频编码。
2. 优先支持硬件编码。
3. 支持为录像模块提供高质量输入。
4. 支持为视觉模块提供实时处理输入。
5. 如需要网络预览，支持输出标准流地址或内部流句柄。

### 4. 流状态监控
1. 监控每路视频的在线状态。
2. 监控实际分辨率、帧率和码率。
3. 记录相机 SDK 返回的关键错误信息。
4. 向平台模块上报当前流状态。

### 5. 与其他模块协作
1. 向 `record-program-module` 提供主机位和辅机位视频输入。
2. 向 `vision-event-module` 提供主机位和辅机位视频输入。
3. 向 `platform-orchestration-module` 提供初始化、启动、停止和状态接口。

---

## 输入
1. 工业相机 SDK
2. 两台 GigE 工业相机
3. 配置文件中的相机参数
4. 平台模块发出的启动与停止命令

---

## 输出
1. 主机位视频输入
2. 辅机位视频输入
3. 流状态信息
4. 相机错误信息
5. 初始化与运行状态

---

## 成功验收标准
1. 能稳定识别并接入两台工业相机。
2. 能持续获取两路视频帧。
3. 能向录像模块和视觉模块提供稳定输入。
4. 相机异常、断流、初始化失败时有明确日志。
5. 平台模块可查询本模块状态。
6. 在单主机集中式架构下可稳定运行。
