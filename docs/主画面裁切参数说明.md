# 主画面裁切参数说明 v1

## 文档信息
- 版本：v1
- 依据：`Team Interface Contract v1.2 Frozen` §3.2（B 模块责任与强制要求）、§8.3（C→B 关注区域与多机位决策）、§9.5（B 模块降级策略）
- 适用模块：record-program-module（录像与主画面生成模块，模块 B）
- 依赖模块：vision-event-module（模块 C，提供关注区域与多机位决策）
- 冻结要求：裁切策略、宽高比、平滑参数一经冻结，MVP 阶段不得大改

---

## 1. 概述

主画面生成模块（record-program-module）负责从双路 5MP（2592×1944）相机输入中裁切出一路 1920×1080 @ 25fps 的转播主画面输出。

裁切流程：
```
5MP 原生帧 (2592×1944)
    → 根据关注区域确定裁切范围
    → 裁切 (expand_then_clip)
    → 缩放到 1920×1080
    → 输出主画面帧
```

---

## 2. 核心裁切参数

| 参数名 | 冻结值 | 说明 |
|--------|--------|------|
| `output_resolution` | `1920×1080` | 主画面输出分辨率 |
| `source_resolution` | `2592×1944` | 相机原生 5MP 分辨率 |
| `fps` | `25` | 输出帧率 |
| `aspect_ratio` | `16:9` | 固定 16:9 输出宽高比 |
| `crop_policy` | `expand_then_clip` | 先扩边到 16:9，再贴边裁切 |
| `smoothing_level` | `light_anti_shake` | 轻微防抖平滑 |
| `min_camera_hold_ms` | `2000` | 最小切镜保持 2 秒 |
| `default_mode` | `follow_multi_focus_regions` | 默认跟随多机位关注区域 |

---

## 3. 裁切策略详解

### 3.1 默认裁切模式

当视觉模块未输出关注区域（初始化/异常/超时）时，使用默认中心裁切：

| 相机 | 默认区域 | 说明 |
|------|------|------|
| cam_01 | `half_field_center_safe_16_9` | 半场中心区域，覆盖中场和两侧禁区 |
| cam_02 | `goal_side_attack_area_16_9` | 门前进攻侧区域 |

### 3.2 动态裁切模式

当视觉模块输出关注区域后，按以下规则裁切：

**规则 1：扩边再裁切（expand_then_clip）**
- 以关注区域为基准，先扩展到 16:9 包围框
- 如果包围框超出画面边界，向内收缩到画面边缘
- 如果关注区域面积过小（宽 < 320px 或 高 < 180px），回退到默认区域

**规则 2：平滑过渡（light_anti_shake）**
- 裁切窗口位置按指数移动平均平滑
- 平滑系数默认 0.7（新位置权重 0.3）
- 防止画面剧烈跳动

**规则 3：最小切镜保持（min_camera_hold_ms = 2000ms）**
- 切换机位后至少保持 2 秒
- 避免频繁来回切镜

### 3.3 多机位决策

视觉模块推送 `program-decision` 消息，指定当前推荐机位：

| recommended_camera_id | 含义 | 裁切行为 |
|------|------|------|
| `cam_01` | 主机位为主 | 从 cam_01 取帧裁切，输出主画面 |
| `cam_02` | 门前区域活跃 | 从 cam_02 取帧裁切，输出主画面 |

**降级策略**：
- 若 `program_decision` 超时未更新（> 2s），回退 `cam_01`
- 若当前推荐机位输入异常（断流/黑屏），立即回退另一路可用机位
- 两路皆不可用时，输出黑屏并告警

## 4. 裁切坐标定义

### 坐标系统

以画面左上角为原点 `(0,0)`，x 向右，y 向下：

```
(0,0) ──────────────────────── (2591, 0)
  │                               │
  │      ┌─────────────┐          │
  │      │  裁切区域    │          │
  │      │ (x, y)      │          │
  │      └─────────────┘          │
  │                               │
(0, 1943) ───────────── (2591, 1943)
```

### 裁切区域字段

```json
{
  "focus_region": {
    "x": 1200,
    "y": 650,
    "width": 1400,
    "height": 800
  }
}
```

| 字段 | 类型 | 范围 | 说明 |
|------|------|------|------|
| `x` | int | 0 ~ 2591 | 裁切区域左上角 x 坐标 |
| `y` | int | 0 ~ 1943 | 裁切区域左上角 y 坐标 |
| `width` | int | > 320 | 裁切区域宽度（像素） |
| `height` | int | > 180 | 裁切区域高度（像素） |

## 5. 边界与异常处理

| 场景 | 处理策略 |
|------|------|
| 关注区域坐标越界 | 裁剪到画面边界内，记录 WARNING |
| 关注区域过小 (w<320 或 h<180) | 回退默认区域，记录 WARNING |
| 视觉模块断连 2s | 回退默认区域，状态标记为 degraded |
| 单路输入断流 | 自动切换到另一路可用机位 |
| 当前机位画面黑屏（平均亮度 < 10） | 尝试切换到另一路机位，记录 ERROR |
| program_decision 超时 | 回退 cam_01 |
| program_decision 置信度 < 0.5 | 保持当前机位，不退回到 cam_01 |

---

## 6. 画面平滑参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| smoothing_alpha | 0.3 | 新目标位置的历史平滑系数（0~1，越小越平滑） |
| max_move_per_frame | 30px | 每帧最大移动像素，防止过度跳变 |
| min_change_to_move | 10px | 变化小于此值时忽略，避免微小抖动 |
| hold_before_switch | 2000ms | 切镜后最短保持时间 |

```json
    "min_change_to_move": 10
  }

```

## 8. 已知限制（MVP 阶段）

| 限制 | 影响 | 缓解方案 |
|------|------|------|
| 不支持特写/广角双镜头切换 | 裁切画面的范围感单一 | 保持机位合理布局 |
| 平滑系数固定 | 不能根据比赛节奏自适应 | 中期可加入动态调整 |
| 最小切镜 2 秒 | 快速攻防转换可能延迟 | 接受 MVP 阶段的滞后 |
| 仅支持 16:9 输出 | 无法适配竖屏或超宽屏 | 当前需求仅 16:9 |
