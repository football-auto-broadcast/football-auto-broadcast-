# 采集与编码分发模块 - 运行说明 v1

## 模块信息
- 模块名：ingest-streaming-module
- 可执行文件：ingest_streaming_service.exe
- 负责人：工程师 A
- 版本：v0.2（第 2 周）

## 依赖环境
- Windows 10/11 x64
- Visual Studio 2022（v143 工具集）
- HIKROBOT MVS SDK（已包含在 third_party/）
- GStreamer 1.28.3（已安装）
- mediamtx v1.8.2（已包含在 bin/）

## 目录结构
services/ingest-streaming-module/
├── src/              # 源码
├── include/          # 头文件
├── bin/              # mediamtx 工具
├── configs/          # 配置文件
└── third_party/      # MVS SDK

## 编译步骤
1. 双击 `ingest-streaming-module.slnx` 用 VS 打开
2. 确认项目属性：
   - 附加包含目录：`$(ProjectDir)include;$(ProjectDir)third_party\MVS SDK\Includes`
   - 附加库目录：`$(ProjectDir)third_party\MVS SDK\win64`
   - 附加依赖项：`MvCameraControl.lib`
3. 选择 Release + x64
4. 按 Ctrl+Shift+B 编译

## 启动步骤（相机到位后）
1. 修改 `configs/ingest-streaming-module/config.json`，填入真实相机序列号
2. 启动 mediamtx：`mediamtx.exe mediamtx.yml`
3. 运行：`ingest_streaming_service.exe`
4. 浏览器访问：`http://127.0.0.1:8081/api/v1/ingest/status`

## 验证方法
- 控制台显示双路 fps、分辨率、在线状态
- ffplay `rtsp://127.0.0.1:8554/main` 能播放画面
- HTTP 接口返回正常 JSON

## 已知限制（第 2 周）
- 当前代码已完成双路相机管理、帧队列、HTTP 接口
- 因硬件不在现场，RTSP 链路用 ffmpeg + 本地视频验证通过
- 相机到位后，修改 config.json 序列号即可直接运行，无需改代码

## 常见问题
- **相机枚举失败**：检查网线、PoE 供电、MVS 客户端能否识别
- **编译找不到头文件**：检查项目属性 → C/C++ → 附加包含目录
- **RTSP 推流失败**：确认 mediamtx 已启动，端口 8554 未被占用
