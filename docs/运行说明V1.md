- Windows 10/11 x64
- Visual Studio 2022（v143 工具集）
- HIKROBOT MVS SDK（已包含在 `third_party/mvs_sdk/`）
- GStreamer 1.28.3（已包含在 `third_party/gstreamer/`）
- MediaMTX v1.8.2（已包含在 `bin/`）

## 编译步骤

1. 打开 `services/ingest-streaming-module/ingest-streaming-module.vcxproj`
2. 确认项目属性：
   - 附加包含目录：`$(ProjectDir)include;$(ProjectDir)third_party\mvs_sdk\Includes`
   - 附加库目录：`$(ProjectDir)third_party\mvs_sdk\win64`
   - 附加依赖项：`MvCameraControl.lib`
3. 选择 **Release + x64**
4. 按 `Ctrl+Shift+B` 编译
5. 编译完成后复制到 bin 目录：

```batch
copy /y "x64\Release\ingest_streaming_service.exe" "bin\ingest_streaming_service.exe"
```

或直接运行 `bin\deploy.bat` 部署全部依赖。

## 启动步骤（实机运行）

**前置条件**：确认相机已通过 PoE 交换机连接，相机指示灯正常。

### 步骤 1：修改配置文件

打开 `configs/ingest_streaming/config.json`，确认相机序列号正确：

- 主相机（cam_01）：`F92514845`
- 辅相机（cam_02）：`D91363830`

### 步骤 2：启动 MediaMTX 双实例

在 `bin/` 目录下，分别启动两个 MediaMTX 实例：

```batch
cd services\ingest-streaming-module\bin
start mediamtx.exe mediamtx_8554.yml
start mediamtx.exe mediamtx_8555.yml
```

或直接运行 `start.bat` 一次性启动所有服务。

### 步骤 3：运行采集模块

```batch
cd services\ingest-streaming-module\bin
ingest_streaming_service.exe
```

## 验证方法

### 方法 1：控制台日志

启动后观察控制台输出，正常状态示例：

## 常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 相机初始化失败 "Camera X not found" | 序列号与实际不匹配 | 核对 MVS 客户端中相机序列号，修改 config.json |
| MediaMTX 端口被占用 | 8554/8555 已被占用 | `netstat -ano \| findstr ":8554"` 查看并关闭占用进程 |
| 相机 online=false | 网线断开或 PoE 供电异常 | 检查网线、交换机、相机指示灯 |
| streaming=false 但 online=true | streamer 未初始化（多为首次启动时相机未就绪） | 等待 10 秒自动重连，或检查 MediaMTX 是否正常运行 |
| 画面马赛克或模糊 | 分辨率缩放问题 | **v1.3 已修复**：当前输出为原生 2592x1944，不再缩放 |
| 推流推送失败 | MediaMTX 未启动 | 先启动两个 mediamtx 实例，再启动采集模块 |
| HTTP 无法访问 | 防火墙阻止或端口占用 | 检查 8081 端口，`netsh advfirewall firewall add rule ...` |
| 程序启动后立即退出 | 配置文件路径错误 | 检查 `configs/ingest_streaming/config.json` 是否存在 |

## 已知限制（第 3-5 周）

- 相机初始化失败后，若后续重新接入，不会自动初始化；需等待程序内置重连机制（约10秒间隔）
- 双实例 MediaMTX 需要分别占用 8554/8555 两个端口，不能合并为单个实例
- HTTP 接口仅支持 GET 查询，暂不支持 PUT/POST 控制命令
- 相机帧率由硬件决定（约24fps），程序内无法强制超频
- GStreamer pipeline 在 Windows 下 CPU 占用较高（x264纯软编码），高分辨率下注意散热
