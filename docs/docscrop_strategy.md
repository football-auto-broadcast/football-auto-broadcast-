\# 主画面裁切参数说明 v1

\*\*模块：\*\* B (record-program-module)

\*\*最后更新：\*\* 2026-04-19



\## 1. 基础分辨率映射

\*   \*\*输入源 (模块 A)\*\*: 4K 级别 (如 3840x2160 或相似的工业相机原生分辨率)。

\*   \*\*输出流 (主画面)\*\*: 固定 1080P (1920x1080)。

\*   \*\*裁切框尺寸\*\*: 固定 1920x1080，不允许形变拉伸。



\## 2. 归一化坐标计算策略

为避免模块 A 切换分辨率导致裁切框偏移，模块 B \*\*强制依赖\*\* 模块 C 提供的归一化坐标：

\*   `focus\_region.x\_ratio` (0.0 \~ 1.0)

\*   `focus\_region.y\_ratio` (0.0 \~ 1.0)

\*   \*\*像素坐标转换公式\*\*:

&#x20;   \*   `Target\_X = x\_ratio \* 输入视频宽度`

&#x20;   \*   `Target\_Y = y\_ratio \* 输入视频高度`



\## 3. 画面平滑算法 (EMA)

为防止“跟球”时镜头出现机械式抖动，采用指数移动平均（EMA）算法模拟真人推流：

\*   \*\*平滑系数 ($\\alpha$)\*\*: `0.15`

\*   \*\*刷新公式\*\*: `Current\_Pos = Current\_Pos + (Target\_Pos - Current\_Pos) \* 0.15`

\*   \*\*边界保护\*\*: 计算出的 `Current\_Pos` 若导致 1920x1080 的裁切框超出输入源的物理边界，必须执行 Clamp (钳位) 截断，防止 OpenCV 越界崩溃。



\## 4. 延迟补偿机制

\*   解析 C 模块传来的 `origin\_timestamp\_ms`。

\*   若计算出：`当前系统时间 - origin\_timestamp\_ms > 150ms`，则认为 AI 坐标已滞后。

\*   \*\*补偿方案\*\*: 内部激活简易线性外推预测（后续版本升级为 Kalman Filter），根据过去 5 帧的运动向量叠加偏移量，再代入 EMA 算法。
