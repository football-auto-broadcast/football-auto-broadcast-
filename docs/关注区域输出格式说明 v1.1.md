# 关注区域输出格式说明 v1.1
**文档版本**: v1.1  
**最后更新**: 2026-04-22  
**适用模块**: `vision-event-module` → `record-program-module`  
**适用架构**: Windows 原生单主机集中式架构  
**适用场景**: 双机位可裁切、多机位画面可用性

---

## 1. 文档目的
本文档定义足球赛事自动转播系统中**双机位关注区域（Focus Regions）** 的 JSON 格式规范。  
视觉分析模块通过此格式向录像与主画面生成模块推送双机位裁切建议，用于：

1. 生成 `cam_01` 裁切画面  
2. 生成 `cam_02` 裁切画面  
3. 为主画面选择提供基础视觉依据

---

## 2. 核心原则
当前版本冻结以下原则：

1. `cam_01` 必须输出可用关注区域  
2. `cam_02` 必须输出可用关注区域  
3. 两路机位都必须具备独立裁切价值  
4. 当前推荐以**批量推送双机位关注区域**为标准接口  
5. 多机位决策信息与双机位关注区域配套使用

---

## 3. 单路关注区域结构
### 3.1 JSON 格式
```json
{
  "camera_id": "cam_01",
  "focus_region": {
    "x": 1200,
    "y": 650,
    "width": 1400,
    "height": 800
  },
  "source_type": "motion_cluster",
  "confidence": 0.87
}
```



### 3.2 字段说明
| 字段名 | 类型 | 必填 | 约束 | 说明 |
| --- | --- | --- | --- | --- |
| `camera_id` | string | 是 | `cam_01`<br/> 或 `cam_02` | 来源机位 |
| `focus_region.x` | integer | 是 | `>= 0` | 左上角 X |
| `focus_region.y` | integer | 是 | `>= 0` | 左上角 Y |
| `focus_region.width` | integer | 是 | `> 0` | 区域宽度 |
| `focus_region.height` | integer | 是 | `> 0` | 区域高度 |
| `source_type` | string | 是 | `ball_detection`<br/>、`motion_cluster`<br/>、`default` | 来源类型 |
| `confidence` | number | 是 | `[0,1]` | 置信度 |


---

## 4. 双机位关注区域集合结构
### 4.1 JSON 格式
```json
{
  "match_id": "match_20260405_001",
  "timestamp_ms": 1712323200123,
  "regions": [
    {
      "camera_id": "cam_01",
      "focus_region": {
        "x": 1200,
        "y": 650,
        "width": 1400,
        "height": 800
      },
      "source_type": "motion_cluster",
      "confidence": 0.87
    },
    {
      "camera_id": "cam_02",
      "focus_region": {
        "x": 320,
        "y": 180,
        "width": 900,
        "height": 600
      },
      "source_type": "ball_detection",
      "confidence": 0.91
    }
  ]
}
```

### 4.2 字段说明
| 字段名 | 类型 | 必填 | 约束 | 说明 |
| --- | --- | --- | --- | --- |
| `match_id` | string | 是 | 非空 | 比赛唯一标识 |
| `timestamp_ms` | integer | 是 | `>= 0` | 毫秒时间戳 |
| `regions` | array | 是 | 长度必须为 2 | 双机位关注区域集合 |


### 4.3 冻结规则
1. `regions` 必须同时包含 `cam_01` 与 `cam_02`
2.  不允许只推一路 
3.  不允许出现第三路机位 
4. `camera_id` 在 `regions` 内必须唯一 

---

## 5. 来源类型枚举
当前冻结为以下 3 个值：

| source_type | 说明 |
| --- | --- |
| `ball_detection` | 基于足球检测 |
| `motion_cluster` | 基于运动聚集 |
| `default` | 默认回退区域 |


### 冻结说明
当前版本不允许新增：

+ `player_cluster`
+ `goal_line_projection`
+  其他自定义值 

若后续扩展，必须升级文档版本。

---

## 6. 坐标系统说明
### 6.1 坐标原点
+  原点位于该路原始帧左上角 
+  X 轴向右为正 
+  Y 轴向下为正 
+  单位为像素 

### 6.2 重要说明
`cam_01` 的关注区域坐标基于 `cam_01` 自身帧尺寸。  
`cam_02` 的关注区域坐标基于 `cam_02` 自身帧尺寸。

两路坐标系彼此独立，不能混用。

---

## 7. HTTP 推送接口
### 7.1 冻结接口
**目标模块**: `record-program-module`  
**目标端口**: `8082`

**请求方法**: `POST`  
**请求路径**:

```plain
/api/v1/record/matches/{match_id}/focus-regions
```

### 7.2 请求体
请求体即第 4.1 节定义的双机位关注区域集合 JSON。

### 7.3 成功响应
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "received": true,
    "applied": true
  }
}
```

### 7.4 失败响应
```json
{
  "code": 1001,
  "message": "invalid focus regions: cam_02 region missing",
  "data": {}
}
```

## 8. 多机位决策配套结构
### 8.1 JSON 格式
```json
{
  "match_id": "match_20260405_001",
  "timestamp_ms": 1712323200123,
  "recommended_camera_id": "cam_02",
  "reason": "goal_area_activity_boosted",
  "confidence": 0.91
}
```

### 8.2 推送接口
**请求方法**: `POST`  
**请求路径**:

```plain
/api/v1/record/matches/{match_id}/program-decision
```

### 8.3 reason 枚举
| reason | 说明 |
| --- | --- |
| `global_play_tracking` | 适合全场主机位 |
| `goal_area_activity_boosted` | 门前活动增强 |
| `six_yard_box_highlight` | 小禁区精彩瞬间 |
| `default_main_camera` | 默认主机位策略 |
| `aux_camera_fallback` | 辅机位备用策略 |

## 10. 校验规则
### 10.1 必填字段校验
必须包含：

+ `match_id`
+ `timestamp_ms`
+ `regions`

每个 region 必须包含：

+ `camera_id`
+ `focus_region`
+ `source_type`
+ `confidence`

`focus_region` 内必须包含：

+ `x`
+ `y`
+ `width`
+ `height`

### 10.2 值域校验
#### camera_id
必须属于：

+ `cam_01`
+ `cam_02`

#### regions
必须满足：

1.  长度为 2 
2.  同时包含 `cam_01` 与 `cam_02`

#### source_type
必须属于：

+ `ball_detection`
+ `motion_cluster`
+ `default`

#### 区域值
必须满足：

```plain
x >= 0
y >= 0
width > 0
height > 0
```

#### confidence
必须满足：

```plain
0 <= confidence <= 1
```

---

## 11. 推荐实现说明
以下内容属于推荐实现，不属于协议硬约束：

1. `cam_01` 默认区域可采用中心 70% 裁切 
2. `cam_02` 默认区域可根据门前视野做单独配置 
3.  B 模块可对两路关注区域分别做平滑 
4.  若某一路关注区域过期，可仅回退该路默认区域，不影响另一机位

4. `cam_01 + cam_02` 必须同时存在的规则 
5. `source_type` 枚举
