# 统一配置说明 v1

## 文档信息
- 版本：v1
- 依据：`Team Interface Contract v1.2 Frozen` §4（进程名/端口）、§5（目录契约）、§6（通信总原则）
- 适用模块：全部 5 个模块（A~E）
- 配置管理责任：platform-orchestration-module（模块 E）
- 冻结要求：配置结构、字段名、加载逻辑冻结后各模块不得自行定义新格式

---

## 1. 概述

系统所有模块通过统一的配置文件格式进行参数管理。配置文件由平台模块统一加载并分发到各模块，禁止各模块自行创建第二套配置体系。

---

## 2. 配置文件位置（合同 §5.2 冻结目录结构）

合同 §6.4 禁止：在代码中写死路径、端口、IP；绕开 E 模块自行定义任务调度协议。

```
{project_root}\configs\
├── system.json                    # 系统全局配置
├── ingest_streaming\config.json   # 采集模块配置
├── record_program\config.json     # 录像模块配置
├── vision_event\config.json       # 视觉模块配置
├── highlight\config.json          # 集锦模块配置
└── platform\config.json           # 平台模块配置
```

---

## 3. 全局系统配置 (system.json)

合同 §4.1 进程名冻结，§4.2 HTTP 端口冻结，§4.3 RTSP 端口冻结，§5.1 数据根目录冻结。

```json
{
  "data_root": "D:\\football\\data",
  "log_level": "info",
  "log_dir": "{data_root}\\logs",
  "disk_min_free_gb": 10,
  "modules": {
    "ingest_streaming": true,
    "record_program": true,
    "vision_event": true,
    "highlight_generation": true,
    "platform_orchestration": true
  },
  "host": "127.0.0.1",
  "ports": {
    "platform": 8080,
    "ingest": 8081,
    "record": 8082,
    "vision": 8083,
    "highlight": 8084
  },
  "rtsp_ports": {
    "cam_01": 8554,
    "cam_02": 8555,
    "program_preview": 8560
  }
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `data_root` | string | 数据根目录（所有录像、元数据、输出根路径） |
| `log_level` | string | 日志级别：debug / info / warn / error |
| `log_dir` | string | 日志目录（支持 {data_root} 变量替换） |
| `disk_min_free_gb` | int | 磁盘最低剩余空间告警阈值（GB） |
| `modules` | object | 各模块启用开关 |
| `host` | string | 本机监听地址 |
| `ports` | object | 各模块 HTTP 端口分配 |
| `rtsp_ports` | object | 各机位 RTSP 端口分配 |

## 8. 配置加载规则

### 8.1 加载顺序

1. 平台模块启动，加载 `system.json`
2. 平台模块根据 `system.json` 中的 `modules` 配置，按需加载各子模块配置
3. 平台模块将相应配置通过 HTTP API 分发给各模块

### 8.2 变量替换

配置中支持 `{data_root}` 变量，在加载时替换为 `system.json` 中的实际值。

### 8.3 配置校验

| 校验项 | 说明 |
|------|------|
| 端口冲突检查 | 各模块 HTTP 端口不得重复 |
| 路径存在性检查 | data_root 必须存在且可写 |
| 磁盘空间检查 | 启动时检查剩余空间 ≥ disk_min_free_gb |
| 必填字段检查 | 缺少必填字段拒绝启动 |

### 8.4 禁止事项

1. 禁止模块自行创建新的配置文件
2. 禁止模块绕过平台模块自行加载配置
3. 禁止在代码中硬编码路径、端口、IP
4. 禁止运行时修改配置而不通过平台模块

---

## 9. 配置变更流程

1. 修改对应 `config.json` 文件
2. 通过平台模块 API 重新加载配置：`POST /api/v1/platform/config/reload`
3. 平台模块校验新配置合法性
4. 平台模块将变更推送到受影响模块
5. 模块需支持热加载（若当前正在录制，部分参数需延后生效）
