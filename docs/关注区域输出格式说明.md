# 关注区域输出格式说明 v1

## 文档信息
- 版本：v1
- 依据：`Team Interface Contract v1.2 Frozen` §8.3（C→B 接口）、§3.3（C 模块强制要求）
- 适用模块：vision-event-module（视觉分析模块，模块 C）
- 消费模块：record-program-module（录像与主画面生成模块，模块 B）
- 冻结要求：输出格式、推送频率、字段语义一经冻结，MVP 阶段不得大改

---

## 1. 概述

视觉分析模块持续分析两路视频画面，为录像与主画面生成模块提供关注区域和多机位决策。

- **关注区域**：画面中当前比赛热点位置，供主画面裁切
- **多机位决策**：推荐当前使用哪个机位的画面作为主画面输出

---

## 2. 关注区域输出

### 2.1 推送频率

频率冻结为 **200ms 一次**（每秒 5 次）。

### 2.2 接口

**方法**：`POST`
**路径**：`/api/v1/record/matches/{match_id}/focus-regions`

### 2.3 请求体结构

```json
{
  "match_id": "match_20260405_001",
  "timestamp_ms": 1712323200123,
  "regions": [
    {
      "camera_id": "cam_01",
      "focus_region": {
        "x": 1200,
        "y": 650,
        "width": 1400,
        "height": 800
      },
      "source_type": "motion_cluster",
      "confidence": 0.87
    },
    {
      "camera_id": "cam_02",
      "focus_region": {
        "x": 320,
        "y": 180,
        "width": 900,
        "height": 600
      },
      "source_type": "ball_detection",
      "confidence": 0.91
    }
  ]
}
```

### 2.4 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `match_id` | string | 是 | 比赛标识符 |
| `timestamp_ms` | int64 | 是 | 本帧对应的系统时间戳（毫秒） |
| `regions` | array | 是 | 双机位关注区域数组，必须包含 cam_01 和 cam_02 |
| `regions[].camera_id` | string | 是 | `cam_01` 或 `cam_02` |
| `regions[].focus_region.x` | int | 是 | 关注区域左上角 x 坐标（像素） |
| `regions[].focus_region.y` | int | 是 | 关注区域左上角 y 坐标（像素） |
| `regions[].focus_region.width` | int | 是 | 关注区域宽度（像素），必须 > 0 |
| `regions[].focus_region.height` | int | 是 | 关注区域高度（像素），必须 > 0 |
| `regions[].source_type` | string | 是 | 检测来源类型 |
| `regions[].confidence` | float | 是 | 置信度，范围 [0.0, 1.0] |

### 2.5 source_type 枚举

| 值 | 含义 | 使用场景 |
|------|------|------|
| `ball_detection` | 足球位置检测 | 球已稳定跟踪时使用 |
| `motion_cluster` | 运动聚集区域 | 球丢失时，以运动最密集区域为替代 |
| `player_cluster` | 球员密集区 | 禁区内多人争抢时使用 |
| `goal_area_activity` | 门前活跃区 | 射门/门前混战时使用 |
| `default_safe_region` | 默认安全区域 | 所有检测器不可用时回退 |

### 2.6 坐标系统

以画面左上角为原点 `(0,0)`，x 向右增大，y 向下增大。

- 参考分辨率：2592×1944（相机原始 5MP）

## 3. 多机位决策输出

### 3.1 推送频率

只在推荐机位变化、`reason` 变化或置信度跨阈值（0.5）变化时推送。

### 3.2 接口

**方法**：`POST`
**路径**：`/api/v1/record/matches/{match_id}/program-decision`

### 3.3 请求体结构

```json
{
  "match_id": "match_20260405_001",
  "timestamp_ms": 1712323200123,
  "recommended_camera_id": "cam_02",
  "reason": "goal_area_activity_boosted",
  "confidence": 0.91
}
```

### 3.4 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `match_id` | string | 是 | 比赛标识符 |
| `timestamp_ms` | int64 | 是 | 决策时间戳（毫秒） |
| `recommended_camera_id` | string | 是 | `cam_01` 或 `cam_02` |
| `reason` | string | 是 | 切换原因 |
| `confidence` | float | 是 | 决策置信度，范围 [0.0, 1.0] |

### 3.5 reason 枚举

| 值 | 含义 |
|------|------|
| `goal_area_activity_boosted` | 门前活跃度提升，推荐使用门前机位 |
| `corner_attack_active` | 角球/边路进攻，推荐使用底线机位 |
| `midfield_dominant` | 中场控球为主，推荐使用主机位 |
| `default_main_camera` | 默认主机位覆盖 |
| `aux_camera_degraded` | 辅机位异常，强制主机位 |
| `main_camera_degraded` | 主机位异常，强制辅机位 |
| `confidence_high_switch` | 高置信度区域从辅机位产生 |

## 4. 关注区域可选落盘格式

视觉模块可将关注区域时间序列落盘为 JSON 文件，用于后续分析和 debug：

`{data_root}\metadata\{match_id}\focus_regions.json`

```json
{
  "match_id": "match_20260405_001",
  "resolution": "2592x1944",
  "update_interval_ms": 200,
  "entries": [
    {
      "timestamp_ms": 1712323200000,
      "cam_01": {
        "focus_region": { "x": 1200, "y": 650, "width": 1400, "height": 800 },
        "source_type": "motion_cluster",
        "confidence": 0.87
      },
      "cam_02": {
        "focus_region": { "x": 320, "y": 180, "width": 900, "height": 600 },
        "source_type": "ball_detection",
        "confidence": 0.91
      }
    }
  ]
}
```

## 5. 强制规则（对应合同 §8.3 冻结规则）

1. `regions` 数组中必须包含 `cam_01` 和 `cam_02` 两条记录
2. 单路 confidence 必须在 `[0, 1]` 之间
3. `width` 和 `height` 必须 > 0
4. 若坐标越界（x+w > 2592 或 y+h > 1944），B 模块自动裁剪至合法范围并记录 WARNING（合同 §8.3 冻结规则 #4）
5. 若关注区域过小（宽 < 320px 或高 < 180px），B 模块回退默认裁切区域（合同 §8.3 冻结规则 #5）
6. 若 `program_decision` 超时，B 模块必须回退到默认主机位策略（合同 §8.3 冻结规则 #6）
7. C 模块必须按 200ms 频率推送双机位关注区域（合同 §8.3 冻结规则 #7）
8. C 模块只在推荐机位变化/push reason 变化/置信度跨阈值变化时才推送 program_decision（合同 §8.3 冻结规则 #8）

---

## 6. 已知限制

| 限制 | 说明 |
|------|------|
| 关注区域计算基于 2592×1944 坐标系 | 若相机以其他分辨率取流需转换 |
| 足球检测在当前 MVP 中可能不稳定 | 运动聚集区域（motion_cluster）为兜底方案 |
| 多机位决策当前仅支持两种机位推荐 | 不处理三机位及以上场景 |
