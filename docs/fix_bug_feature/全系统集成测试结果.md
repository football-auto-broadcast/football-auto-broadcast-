## 本次修复摘要

### A 模块
- 修复 `MV_E_CALLORDER`：枚举提升到 IngestEngine 全局一次（移除双重枚举）
- 文件：`camera_device.h/cpp`, `ingest_engine.cpp`

### B 模块
- 修复 `stop()` 挂死：preview pipe 先关闭再 join worker，加 5 秒超时 + detach 兜底
- 启动 MediaMTX 8560 实例（program preview RTSP 目标）
- 文件：`record-program-module/src/main.cpp`

### C 模块
- 实现 `FFmpegFrameReader`：独立 FFmpeg 子进程 + 独立拉流线程，替换阻塞式 OpenCV VideoCapture
- 帧时间戳使用 reader 捕获时间（system_clock），不用消费者时间
- 占位信号在读帧后立即设置，不等 YOLO 推理完成
- YOLO 推理异步化：独立线程执行，不阻塞帧读取主循环
- 文件：`ffmpeg_frame_reader.hpp/cpp`, `service.cpp`, `CMakeLists.txt`, `default_config.yaml`

### D 模块
- 未修改。highlight 生成故障待后续单独排查

### E 模块
- 未修改

## 已知遗留问题

1. **D 模块 highlight 生成失败**: Pipeline 内部 FFmpeg 处理错误（code=1006）。需排查 `pipeline.cpp` 中的 FFmpeg 命令
2. **cam_02 间歇 offline**: YOLO 异步化后占位符持续更新，但 cam_02 偶现 offline（t+20,25s）。可能为 FFmpeg 子进程重连导致的时间窗口
3. **record_index.json rec_dur 显示负数**: B 模块系统状态 API 中 `record_duration_sec` 字段取反，需检查时间戳计算

## 运行说明

详见 [docs/start/启动.md](../start/启动.md)

### 关键启动命令

```powershell
# 启动 MediaMTX 三实例 (8554/8555/8560)
cd services\ingest-streaming-module\bin
start mediamtx.exe mediamtx_8554.yml
start mediamtx.exe mediamtx_8555.yml
start mediamtx.exe mediamtx_8560.yml

# 启动 A 模块
.\ingest_streaming_service.exe

# 启动 B 模块
cd services\record-program-module\build
.\record_program_service.exe

# 启动 C 模块（注意 WorkingDirectory！）
cd services\vision-event-module\build-ninja-msvc
.\vision_event_service.exe

# 启动 D 模块
cd services\highlight-generation-module\build
.\highlight_generation_service.exe

# 启动 E 模块
cd services\platform-orchestration-module\build\bin
.\platform_orchestration_service.exe
```
