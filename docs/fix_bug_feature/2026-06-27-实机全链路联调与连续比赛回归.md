## 本轮修复

### C 模块

- 将 FFmpeg rawvideo 匿名管道缓冲从系统默认值提升到 1 MB，解决 1080p BGR24 每帧约 6.2 MB 时的严重吞吐下降。
- 删除主循环中遗留的 cam_02 同步推理，避免阻塞拉流循环。
- YOLO worker 支持任一路新帧触发，不再要求双路在同一 40 ms tick 到达。
- degraded 恢复条件改为双路最近帧均未超过 stale timeout，不再要求同 tick 消费。

修复前每路约 3-4 秒才组装一帧；修复后 35 秒比赛中每路组装超过 300 帧，最近帧年龄稳定在约 40-110 ms，C 状态恢复为 running。

### B 模块

- program MP4 和 RTSP preview 编码器由 `_popen/_pclose` 改为 Win32 `CreateProcess + stdin pipe + process handle`。
- stop 改为实际 join program worker，删除通过 `joinable()` 轮询并最终 detach 的错误逻辑。
- 编码器 stop 会关闭 stdin、等待 FFmpeg 完成封装，超时后再终止进程。

修复了第一场结束后，第二场 start 阻塞在 preview `_popen`、HTTP 8082 全部超时的问题。
