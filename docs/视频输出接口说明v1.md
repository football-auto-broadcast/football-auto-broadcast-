## 1. 视频输出规范（冻结）

### 1.1 RTSP流输出

| 机位 | 冻结RTSP地址 | MediaMTX端口 | 分辨率 | 编码 |
|------|-------------|-------------|--------|------|
| 主机位（cam_01） | `rtsp://127.0.0.1:8554/main` | 8554 | 2592x1944 @24fps | H.264 |
| 辅机位（cam_02） | `rtsp://127.0.0.1:8555/aux` | 8555 | 2592x1944 @24fps | H.264 |

> **分辨率说明**：采用海康威视 GigE 工业相机原生分辨率 2592x1944（4:3），保持原始比例，避免画面变形。帧率以实际相机输出为准（当前约24fps）。

### 1.2 编码参数（冻结）

- 编码格式：H.264（x264编码器）
- 分辨率：**2592x1944**（保持相机原生4:3，不缩放至16:9）
- 帧率：~24fps（由相机硬件决定）
- 像素格式：I420（YUV 4:2:0）
- 传输协议：RTSP over TCP
- x264参数：
  - `bitrate=20000`（20Mbps高码率）
  - `speed-preset=medium`（优于ultrafast）
  - `qp-min=10 / qp-max=30`（质量范围约束）
  - `tune=zerolatency`（低延迟）
  - `key-int-max=25`（I帧间隔约1秒）
  - `config-interval=1`（每个IDR帧携带SPS/PPS）

### 1.3 GStreamer Pipeline

```
appsrc (RGB24 2592x1944@24fps)
  → videoconvert
  → video/x-raw (I420 2592x1944@24fps)
  → x264enc (tune=zerolatency, bitrate=20000, qp-min=10, qp-max=30)
  → h264parse (config-interval=1)
  → rtspclientsink (protocols=tcp, latency=0)
```

## 2. HTTP控制接口（冻结）

### 2.1 基础信息

- 模块HTTP端口：`8081`
- 监听地址：`127.0.0.1`（仅本地访问，不对外暴露）
- 访问地址：`http://127.0.0.1:8081`

### 2.2 通用响应格式

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

### 2.3 状态查询接口

**GET** `/api/v1/ingest/status`

响应示例：
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "status": "running",
    "camera_count": 2,
    "online_count": 2,
    "cameras": [
      {
        "camera_id": "cam_01",
        "role": "main",
        "online": true,
        "streaming": true,
        "frame_count": 1240,
        "width": 2592,
        "height": 1944,
        "fps": 24.0
      },
      {
        "camera_id": "cam_02",
        "role": "aux",
        "online": true,
        "streaming": true,
        "frame_count": 1185,
        "width": 2592,
        "height": 1944,
        "fps": 24.0
      }
    ]
  }
}
```

### 2.4 状态字段说明

| 字段 | 说明 |
|------|------|
| `status` | 引擎状态：`idle`/`initializing`/`running`/`degraded`/`stopped`/`failed` |
| `camera_count` | 配置的相机总数 |
| `online_count` | 当前在线的相机数量 |
| `cameras[].camera_id` | **必须为 `cam_01`/`cam_02`**（合同约定字符串格式） |
| `cameras[].role` | 角色：`main`/`aux` |
| `cameras[].online` | 相机SDK连接状态 |
| `cameras[].streaming` | RTSP推流状态 |
| `cameras[].frame_count` | 累计接收帧数 |

## 3. MediaMTX双实例配置

由于双路推流使用不同的RTSP端口，需要启动两个独立的MediaMTX实例：

| 实例 | 配置文件 | RTSP端口 | HLS端口 | RTSP路径 |
|------|---------|---------|---------|---------|
| 实例1（主机位） | `mediamtx_8554.yml` | 8554 | 8888 | `/main` |
| 实例2（辅机位） | `mediamtx_8555.yml` | 8555 | 8889 | `/aux` |

> 每个配置文件通过 `rtspAddress` 字段指定独立端口，通过 `paths.{path}.source: publisher` 声明允许外部推流。

## 4. 快速验证命令

```batch
:: 播放主机位画面
ffplay -rtsp_transport tcp rtsp://127.0.0.1:8554/main

:: 播放辅机位画面
ffplay -rtsp_transport tcp rtsp://127.0.0.1:8555/aux

:: 查询状态
curl http://127.0.0.1:8081/api/v1/ingest/status
```
