# 集锦导出规范说明书 v1.0

**文档版本**: v1.0  
**最后更新**: 2026-05-20  
**适用模块**: `highlight-generation-module` 与 `record-program-module` 联调交接规约  
**适用系统**: Windows 原生运行  
**执行核心**: `third_party/windows/ffmpeg/bin/ffmpeg.exe`

---

## 1. 文档目的

本文档用于死锁模块 D（集锦生成）调用外部 FFmpeg 工具进行视频高速裁剪、无损流拼接、音视频轨道对齐及物理路径落盘的技术指标。本规范严格承接模块 B 的文件索引服务返回格式，确保大规模 4K 高清视频流在集中式主机中剪辑时不需要二次重编码，达到秒级导出的工业级技术标准。

---

## 2. 输入视频流消费规范

模块 D 绝对不允许硬编码写死原始视频输入路径。必须通过调用模块 B 的本地 API 获取物理绝对路径。

### 2.1 依赖交接映射
解析 `GET /api/v1/record/matches/{match_id}/files` 接口返回的 JSON。
- **目标视频消费字段**：读取 `data.program_files` 数组中第 0 个元素的 `file_path`。
- **路径物理限制**：依据 Windows 容器与局域网挂载约定，路径必须形如 `C:\\mnt\\shared\\...` 格式。
- **独占锁释放校验**：模块 D 在读取该 `.mp4` 文件前，必须循环检测文件句柄，确保模块 B 的落盘写入进程已安全退出，严禁发生文件独占冲突（Sharing Violation）。

---

## 3. FFmpeg 裁剪核心参数与时序生成

为了实现极速、无损的裁剪，模块 D 必须强制采用 **Stream Copy (流拷贝)** 模式，禁止重解码视频帧。

### 3.1 核心剪切命令行模板
对于经过筛选层输出的第 $i$ 个高光片段，主程序应通过 C++ `CreateProcess` 或 `std::system` 异步或同步唤醒执行以下底层命令：

```cmd
"C:\\football-auto-broadcast\\third_party\\windows\\ffmpeg\\bin\\ffmpeg.exe" -ss [start_sec] -to [end_sec] -i "[input_file_path]" -c copy -avoid_negative_ts 1 -y "[segment_output_path]"
```

### 3.2 参数严格限定规范

1. `-ss [start_sec]`：必须置于 `-i` 参数**之前**。此举可激活 FFmpeg 的快速关键帧搜寻机制（Input Seeking），将寻道耗时由分钟级降为毫秒级。
2. `-to [end_sec]`：采用绝对终止秒数限制，配合流拷贝，防止视频尾部截取过长。
3. `-c copy`：音视频物理轨道参数物理复制。不允许加入任何 `-vf` 滤镜或改变比特率，确保源转播流的 4K 分辨率、色域及帧率完美保留。
4. `-avoid_negative_ts 1`：强制修正因流拷贝导致的分段音视频时间戳负值问题，防止后续拼接时出现严重的音画不同步。

## 4. 视频拼接与音视频轨道规约 (Stitching Protocol)

各个零散的高光片段裁剪完成后，必须将其缝合为单体大文件。模块 D 禁用常规滤镜拼接，统一锁死使用 **Concat Demuxer (连接复用器)** 技术。

### 4.1 临时清单文件控制

模块 D 必须在临时工作目录内派生出一个名为 `file_list.txt` 的纯文本文件。内容格式严格限定如下（每行代表一个切片物理路径）：

Plaintext

```
file 'C:\\mnt\\shared\\temp\\seg_0001.mp4'
file 'C:\\mnt\\shared\\temp\\seg_0002.mp4'
```

### 4.2 终极拼接命令行

清单文件封装完成后，调度以下命令执行无缝咬合：

DOS

```
"C:\\football-auto-broadcast\\third_party\\windows\\ffmpeg\\bin\\ffmpeg.exe" -f concat -safe 0 -i "C:\\mnt\\shared\\temp\\file_list.txt" -c copy -movflags faststart -y "C:\\mnt\\shared\\output\\highlights\\match_final_highlight.mp4"
```

### 4.3 核心技术参数解释

- `-safe 0`：允许接受 Windows 本地盘符全绝对路径，规避 FFmpeg 默认的安全沙箱拒绝访问。
- `-movflags faststart`：极其关键。它负责将拼接后 MP4 文件的 `moov` 原子（视频元数据索引表）从文件末尾强制挪移至文件头部。**落地效果**：平台调度 E 模块前端在网页渲染播放或校队成员点击时，可实现“即点即播”，无需等待整个 4K 视频完全下载完，大幅提升平台交互体验。

## 5. 目标物理输出规范与环境清理

### 5.1 终产物落盘标准

- **强制输出路径**：`C:\\mnt\\shared\\output\\highlights\\match_final_highlight.mp4`
- **物理规格检查**：输出文件的编码格式必须继承主画面（H.264/H.265），音频轨道格式（AAC）及采样率（44.1kHz/48kHz）不准发生任何形变。

### 5.2 垃圾文件物理清理规约

最终集锦成功产出并写入完毕后，主程序必须执行文件系统扫尾动作：

1. 遍历清理 `temp/` 目录下的所有零散高光片段 `seg_xxxx.mp4`。
2. 删除 `file_list.txt` 临时索引。
3. 释放全部进程句柄，最后通过 `return 0;` 信号通知崔紫林的平台调度模块。
