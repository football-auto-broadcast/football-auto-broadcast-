# 提交集锦生成任务接口说明书 v1.0

**文档版本**: v1.0  
**最后更新**: 2026-05-20  
**适用模块**: `highlight-generation-module` (模块 D 暴露端) → `platform-scheduler-module` (模块 E 消费端)  
**接口类型**: 本地 RESTful HTTP API (基于轻量级异步服务框架)  
**传输协议**: JSON over HTTP (Windows 本地环回网络 `127.0.0.1`)

---

## 1. 结构定义与目的

本接口用于响应平台调度模块（E 模块 —— 崔紫林）发起的集锦生成请求。当一场足球比赛录制结束且视觉模块完成数据归档后，E 模块通过异步拉起本接口，将比赛唯一标识及控制参数注入模块 D，触发后台的 FFmpeg 裁剪与无损拼接流水线。

---

## 2. API 接口规约

### 2.1 请求基础信息
- **请求方法 (Method)**: `POST`
- **请求路径 (URL)**: `/api/v1/highlight/tasks`
- **内容类型 (Headers)**: `Content-Type: application/json`

### 2.2 请求参数说明 (Request Body)
请求体必须为标准的 JSON 格式，字段定义如下：

| 字段名             | 类型      | 是否必填 | 默认值                      | 约束与语义说明                                     |
| :----------------- | :-------- | :------- | :-------------------------- | :------------------------------------------------- |
| `match_id`         | `String`  | **是**   | 无                          | 比赛唯一标识符，用于和 B、C 模块的文件命名空间对齐 |
| `max_duration_sec` | `Integer` | 否       | `180`                       | 生成的高光集锦总时长上限，用于激活降序权重筛选机制 |
| `output_filename`  | `String`  | 否       | `match_final_highlight.mp4` | 最终生成的精彩集锦视频文件名                       |
| `force_overwrite`  | `Boolean` | 否       | `true`                      | 若目标输出路径已存在同名文件，是否强制执行物理覆盖 |

#### 请求示例 (JSON):
```json
{
  "match_id": "bupt_match_20260519",
  "max_duration_sec": 120,
  "output_filename": "post_game_best_moments.mp4",
  "force_overwrite": true
}
```

## 3. 响应结构规范 (Response Body)

模块 D 接收到任务后，将立即校验前置文件是否存在。校验通过后会立即返回 **`202 Accepted`** 状态码并异步启动 FFmpeg 核心线程，防止 E 模块网络连接发生因剪辑耗时引发的 Read Timeout。

### 3.1 成功接收任务响应 (HTTP 202)

JSON

```
{
  "code": 0,
  "message": "Highlight generation task submitted successfully.",
  "data": {
    "task_id": "task_hl_20260519_8891",
    "match_id": "bupt_match_20260519",
    "status": "PROCESSING",
    "submitted_at": "2026-05-19T23:59:59Z",
    "expected_output_path": "C:\\\\mnt\\\\shared\\\\output\\\\highlights\\\\post_game_best_moments.mp4"
  }
}
```

### 3.2 失败中断响应 (HTTP 400 / 500)

若在任务接收入口层发生前置依赖文件破损，直接拦截并返回对应错误码：

#### 场景 A：缺少视觉分析输入事件（对应模块 D 状态码 `1001`）

JSON

```
{
  "code": 1001,
  "message": "Task rejected: Source event file [configs/highlight-generation-module/events.json] not found.",
  "data": {
    "match_id": "bupt_match_20260519",
    "status": "FAILED"
  }
}
```

#### 场景 B：缺少录像索引文件（对应模块 D 状态码 `1002`）

JSON

```
{
  "code": 1002,
  "message": "Task rejected: File index reference [data/records/record_index.json] cannot be resolved.",
  "data": {
    "match_id": "bupt_match_20260519",
    "status": "FAILED"
  }
}
```

## 4. 注意事项与联调规约

1. **异步执行通知**：E 模块在收到 `code: 0` 的响应后，应当启动本地定时器，轮询模块 D 的任务状态接口（或通过物理文件句柄监听），直至进程返回退出码 `0`。
2. **绝对路径隔离**：此处返回的 `expected_output_path` 严格基于全组约定的 `C:\\mnt\\shared\\...` 共享网络盘物理隔离区。E 模块前端后台直接通过操作该路径即可获取文件进行分发。
